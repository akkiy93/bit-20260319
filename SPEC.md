# SPEC.md

## プロジェクト概要

社内勉強会（2026/03/19）向けプレゼン資料の作成。
Claude Codeを使った業務効率化のデモを通じて、AIツールの正しい活用法を伝える。

## 対象聴衆

- SESエンジニア（自社開発ではなく、クライアントから要件を受けて開発する形態）
- Claude Codeに対して「魔法」または「まだ使えない」という両極端な認識を持っている人が混在している

## デモ仕様

### 使用ファイル構成

```
bit-20260319/
├── PLAN.md           # このプロジェクトの目的・流れ
├── SPEC.md           # 本ファイル。仕様・制約の定義
├── TODO.md           # タスク管理
├── KNOWLEDGE.md      # 参考記事の知見まとめ
├── demo/
│   ├── feature_spec.md       # 架空の機能仕様書（ログイン機能想定）
│   ├── test_spec.md          # AIが提案→人間が補完したテスト仕様書
│   ├── convert.py            # Markdown→Excel変換スクリプト
│   └── test_spec.xlsx        # 変換後の成果物
└── slides/
    └── presentation.md       # スライド（Marp形式）
```

### 架空の機能仕様書の内容（feature_spec.md）

- テーマ：ログイン機能（SESでよくある、誰でも理解できる題材）
- 含む要素：機能概要、入力項目定義、バリデーション、エラーケース、画面遷移

### テスト仕様書の構造（test_spec.md）

| 列名 | 内容 |
|------|------|
| テストID | T-001, T-002... |
| テスト区分 | 正常系 / 異常系 / 境界値 |
| テスト項目 | 何をテストするか |
| 操作手順 | どう操作するか |
| 期待結果 | 何が起きれば合格か |
| 確認結果 | 合格 / 不合格（実行時に人間が記入） |
| 備考 | 特記事項 |

### Excel変換スクリプトの仕様（convert.py）

- 入力：test_spec.md
- 出力：test_spec.xlsx
- 使用ライブラリ：openpyxl
- 整形要件：
  - ヘッダー行に背景色（青系）を付ける
  - 列幅を内容に合わせて自動調整
  - テーブル部分はExcelテーブルとして定義
  - 見出し・セクション区切りも反映する

### スライドの仕様

- ツール：Marp（Markdownベース）
- 構成（案）：
  1. タイトル
  2. 今日話すこと
  3. よくある2つの誤解
  4. 正しい使い方（人への依頼と同じ）
  5. デモ紹介：テスト仕様書を作る流れ
  6. デモ①：機能仕様書.md（スクショ）
  7. デモ②：AIのテスト項目提案（スクショ）
  8. デモ③：人間による補完（スクショ）
  9. デモ④：Excel変換結果（スクショ）
  10. まとめ：半自動が最強

## 制約・留意事項

- 実務データは使わない（架空データのみ）
- 作業中にスライドで使えるスクリーンショットを意識して残す
- 「全自動」ではなく「半自動」の価値を示すことが最重要
- スライドは社内向けなので専門用語は最小限に
