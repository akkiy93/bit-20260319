# 機能仕様書：ログイン機能

## 1. 概要

| 項目 | 内容 |
|------|------|
| 機能名 | ログイン機能 |
| 対象システム | 社内業務管理システム |
| バージョン | 1.0.0 |
| 作成日 | 2026/03/15 |
| 作成者 | 開発チーム |

本機能は、登録済みユーザーがIDとパスワードを使って社内業務管理システムにログインするための機能である。

---

## 2. 画面構成

### 2.1 ログイン画面

- URL：`/login`
- 遷移元：任意のページ（未認証時にリダイレクト）
- 遷移先：
  - 認証成功 → ダッシュボード画面（`/dashboard`）
  - 認証失敗 → ログイン画面（エラーメッセージ表示）

### 2.2 入力項目

| 項目名 | 種別 | 必須 | 最大文字数 | 備考 |
|--------|------|------|-----------|------|
| ユーザーID | テキスト | 必須 | 50文字 | 半角英数字・アンダースコアのみ |
| パスワード | パスワード | 必須 | 128文字 | 入力値はマスク表示 |
| ログインボタン | ボタン | - | - | クリックで認証処理を実行 |
| パスワードを忘れた方はこちら | リンク | - | - | パスワードリセット画面へ遷移 |

---

## 3. バリデーション仕様

### 3.1 クライアントサイドバリデーション（送信前）

| 条件 | エラーメッセージ |
|------|----------------|
| ユーザーIDが未入力 | 「ユーザーIDを入力してください」 |
| パスワードが未入力 | 「パスワードを入力してください」 |
| ユーザーIDが使用不可文字を含む | 「ユーザーIDは半角英数字・アンダースコアのみ使用できます」 |

### 3.2 サーバーサイドバリデーション（認証処理）

| 条件 | エラーメッセージ | 処理 |
|------|----------------|------|
| ユーザーIDが存在しない | 「ユーザーIDまたはパスワードが正しくありません」 | ログイン失敗カウント+1 |
| パスワードが一致しない | 「ユーザーIDまたはパスワードが正しくありません」 | ログイン失敗カウント+1 |
| アカウントがロック中 | 「アカウントがロックされています。管理者にお問い合わせください」 | 処理中断 |

---

## 4. アカウントロック仕様

- ログイン失敗 **5回連続** でアカウントをロックする
- ロック後は管理者による手動解除が必要
- ロック状態でのログイン試行はエラーメッセージを表示して処理を中断する（失敗カウントは増加しない）

---

## 5. セキュリティ要件

- パスワードはデータベース上でハッシュ化（bcrypt）して保存する
- 認証成功後はセッションIDを再生成する（セッション固定攻撃対策）
- ログイン試行はすべてログに記録する（ユーザーID・IPアドレス・日時・成否）
- HTTPS通信必須

---

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンスタイム | 認証処理は3秒以内に完了すること |
| 同時接続 | 100ユーザーの同時ログインに対応すること |
| ブラウザ対応 | Chrome・Edge・Firefox の最新版 |
